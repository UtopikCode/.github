## 🚀 Quickspaces: On-Demand Development Environments

Quickspaces is an **open-source, self-hostable alternative to GitHub Codespaces**. It provides development environments that spin up on demand and tear down automatically—no vendor lock-in, no US surveillance, no data dependencies.

### Why Quickspaces?

- **🛡️ Data Sovereignty** — Your code and infrastructure stay under your control, anywhere
- **🌍 Infrastructure Independence** — Run on your own servers, in your country, on your network
- **🔐 No Vendor Lock-In** — Open source and self-hostable—escape US-based SaaS platforms
- **🔧 Docker-Native** — Full Docker Compose support for realistic dev environments
- **☁️ Infrastructure-Agnostic** — Works on AWS, on-premises, TrueNAS, Proxmox, or any infrastructure

### How It Works

1. **Open a repository** with `.devcontainer` config
2. **Click "Create Workspace"** via the VS Code extension
3. **Get a full dev environment** in 1–5 minutes (with database, cache, and all services)
4. **Work normally** — it feels like local development
5. **Pause automatically** when idle — costs stop immediately
6. **Resume later** or destroy it when done

### The Sovereignty Advantage

**Why Self-Hosting Matters**

Many teams must keep their infrastructure independent from US-based corporations:

- **🇪🇺 EU/GDPR Compliance** — Data residency requirements mean your code cannot leave your borders
- **🏛️ Government & Defense** — National security mandates on-premises infrastructure
- **🏢 Enterprise Independence** — Corporations unwilling to rely on US SaaS vendors
- **🌐 Geopolitical Reality** — Not all organizations can trust US cloud platforms

Quickspaces lets you run a complete development environment **entirely within your infrastructure**, with **no data flowing to US companies**.

**The Cost Benefit (Secondary)**

As a bonus: running your own infrastructure is also cheaper. You own the hardware cost once, not billable hours to a SaaS vendor.
