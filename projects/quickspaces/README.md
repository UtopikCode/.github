# Quickspaces Documentation

This directory contains the official Quickspaces documentation collection for the UtopikCode docs repository.

Quickspaces is an **open-source, self-hosted alternative to GitHub Codespaces**. It provides development environments that spin up on demand and tear down automatically—no vendor lock-in, no data dependencies.

## Why Quickspaces?

- **🛡️ Data Sovereignty** — Your code and infrastructure stay under your control
- **🌍 Infrastructure Independence** — Run on your own servers, in your country, on your network
- **🔐 No Vendor Lock-In** — Open source and self-hostable
- **🔧 Docker-Native** — Full Docker Compose support for realistic dev environments
- **☁️ Infrastructure-Agnostic** — Works on AWS, on-premises, TrueNAS, Proxmox, or any infrastructure

## How it works

1. **Open a repository** with `.devcontainer` config
2. **Click "Create Workspace"** via the VS Code extension
3. **Get a full dev environment** in 1–5 minutes
4. **Work normally** — it feels like local development
5. **Pause automatically** when idle — costs stop immediately
6. **Resume later** or destroy it when done

## The sovereignty advantage

**Why self-hosting matters**

- **🇪🇺 EU/GDPR Compliance** — Keep your code in your region
- **🏛️ Government & Defense** — National security requirements on-premises
- **🏢 Enterprise Independence** — Avoid reliance on US SaaS vendors
- **🌐 Geopolitical Reality** — Not all organizations can trust US cloud platforms

Quickspaces lets you run a complete development environment **within your infrastructure**.
